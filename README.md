# Односторінковий додаток (SPA) «Система коментарів»

Сучасний веб-додаток для публікації, відображення та каскадної модерації деревоподібних коментарів. Весь стек та необхідна інфраструктура повністю контейнеризовані за допомогою Docker і запускаються однією командою. 

Проєкт розроблено з акцентом на безпеку, відповідність стандартам XHTML та швидку роботу інтерфейсу в реальному часі.

---

## Технологічний стек

- **Frontend:** React, TypeScript, Vite 
- **Backend API:** Python 3.11+, Django REST Framework (DRF), Daphne (ASGI)
- **Database:** PostgreSQL 15 
- **Caching & Message Broker:** Redis 7 (зберігання ключів капчі, брокер для Celery та Django Channels)
- **Asynchronous Tasks:** Celery (фонова асинхронна обробка та оптимізація зображень)
- **Real-time:** WebSockets через Django Channels (миттєва синхронізація коментарів між клієнтами)
- **Containerization:** Docker, Docker Compose

---

## Реалізований функціонал

1. **Деревоподібна структура коментарів:** Можливість залишати каскадні відповіді на будь-які коментарі без обмеження глибини вкладеності. Реалізовано гнучке сортування за датою додавання та даними автора.
2. **XHTML валідація:** Текст коментарів проходить повну перевірку на рівні бекенду за допомогою `xml.etree.ElementTree`. Якщо користувач ввів незакриті або неправильно вкладені теги (`<a>`, `<code>`, `<i>`, `<strong>`), система повертає валідаційну помилку, захищаючи інтерфейс від поломки розмітки.
3. **Безпека та XSS-захист:** Використання бібліотеки `bleach` для тотальної фільтрації введеного HTML/XHTML тексту. Будь-які небезпечні теги або скрипт-ін'єкції вичищаються перед збереженням в БД.
4. **Вбудований WYSIWYG Редактор:** Зручні кнопки форматування (`Жирний`, `Курсив`, `Код`, `Посилання`) над полем введення тексту. Текст коду автоматично стилізується акуратним моноширинним шрифтом `Courier New`.
5. **Попередній перегляд:** Можливість миттєво побачити фінальний рендеринг коментаря (із застосованими тегами) перед його фактичною відправкою на сервер.
6. **Безпечна Капча:** Динамічна генерація картинок капчі. Значення записуються у швидку пам'ять Redis з механізмом моментального видалення після першої спроби введення, що повністю блокує автоматизований спам.
7. **Асинхронна обробка зображень:** Прикріплені картинки обробляються фоновим воркером Celery за допомогою бібліотеки `PIL (Pillow)`. Великі зображення пропорційно стискаються до заданого ліміту $320 \times 240$ пікселів, знижуючи навантаження на мережу.
8. **Валідація файлів та Lightbox:** Текстові файли (`.txt`) обмежуються розміром до 100 Кб. Для графічних файлів реалізовано інтерактивну галерею Lightbox - клік по зображенню у гілці коментарів плавно відкриває його на весь екран.
9. **JWT Авторизація для гостей:** Система автоматично зберігає ідентифікаційні дані в `localStorage`. При повторному візиті користувачеві не потрібно заново вводити нікнейм, пошту чи сайт.

## Порівняння проектування (Django Models) та реалізації (PostgreSQL DDL)

Для демонстрації повної відповідності архітектурного задуму та фінальної реалізації фізичної бази даних, у каталозі `./docs/schema.sql` згенеровано універсальний та валідний файл схеми (DDL). 

**Зіставлення моделей та фізичних таблиць:**
- **Сутність `User`:** У коді спроектовано кастомну модель користувача (спадкоємець `AbstractUser`) з унікальними індексами на `username` та `email`. У фізичній СУБД це реалізовано таблицею з обмеженнями `VARCHAR NOT NULL UNIQUE` та автоматичним побудуванням швидких B-Tree індексів для пошуку.
- **Сутність `Comment`:** Спроектовано зв'язок `ForeignKey` на автора та рекурсивне поле `parent` (на саму себе). У файлі `schema.sql` це чітко відображено через первинний ключ `BIGINT GENERATED BY DEFAULT AS IDENTITY`, каскадне обмеження зовнішнього ключа (`REFERENCES public.users`), та self-referencing зовнішній ключ `parent_id`, який дозволяє значення `NULL` для кореневих коментарів проєкту.

Для повного розгортання всього оточення, фронтенду, API-сервера та баз даних на хостингу або локальній машині вам знадобиться лише встановлений **Docker** та **Docker Compose**.

1. Клонуйте репозиторій та перейдіть у корінь проєкту:
   ```bash
   git clone https://github.com/Vitaliy250507/TestTask.git
   cd TestTask
2. Запустіть автоматичне збирання та підняття всіх мікросервісів однією командою:
    ```bash
    docker-compose up --build
3. Після завершення збирання система автоматично запустить та зв'яже між собою:
    Frontend (React + Vite) — http://localhost:5173
    Backend API (Django REST Framework) — http://localhost:8000
    Database (PostgreSQL) — порт 5432
    Cache & Broker (Redis) — порт 6379
    Task Queue (Celery Worker) — фоновий моніторинг завдань
